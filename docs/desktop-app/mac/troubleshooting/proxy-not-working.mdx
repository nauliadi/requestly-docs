# Requestly Proxy not working

Steps for Troubleshooting Requestly System wide proxy on OSX

### Check Certificate[](#check-certificate)

For Requestly to intercept HTTPS traffic, you need to install Requestly Certificate in the keychain

- Open keychain access (Press cmd + enter and then type "keychain")
- Select login keychain
- Check whether **RequestlyCA** is present and trusted or not.

If certificates are not installed & trusted correctly, then follow the steps [here](/desktop-app/how-to.../osx-setup#setup-certificate).

### Check Proxy[](#check-proxy)

- Navigate to System Preferences > Network > Advanced > Proxies
- Click on **Web Proxy (HTTP)**
  - Check whether the settings are set to **127.0.0.1** & **8080** (Port might be different. Check which port Requestly proxy is running on from top right of desktop app
  - Check whether it is enabled or not
- Click on Secure Web Proxy (HTTPS)
  - Check whether the settings are set to **127.0.0.1** & **8080** (Port might be different. Check which port Requestly proxy is running on from top right of desktop app)
  - Check whether it is enabled or not
